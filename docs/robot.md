# Robot Entity

## Purpose

A Robot represents a physical, planned, or prototype humanoid robotic system.

The Robot entity is the central object in Humanoid Intelligence Core.

## Examples

- Tesla Optimus
- Unitree G1
- Figure 01
- Apptronik Apollo
- Agility Digit

## Required Fields

- Robot name
- Manufacturer
- Country
- Status
- Year introduced

## Core Specifications

- Height
- Weight
- Payload
- Speed
- Degrees of freedom
- Runtime
- Battery type
- Charging time

## Intelligence Fields

- Mobility maturity
- Manipulation maturity
- AI autonomy
- Commercial readiness
- Developer friendliness
- Deployment readiness

Scale: 1–5

## Relationships

A Robot can connect to:

- Manufacturer
- Country
- Technologies
- Components
- Applications
- Companies
- People
- Events
- News
- Competitors

## Design Rules

1. A robot must belong to one manufacturer.
2. A robot should have one primary country.
3. Specifications should be stored as structured data, not text.
4. Every major claim should have a source.
5. A robot profile should be useful even without a news article.
6. A robot can be compared with other robots using shared fields.

## Future Use

Robot data will power:

- Robot profiles
- Comparison tables
- Search filters
- Rankings
- Country pages
- Manufacturer pages
- Intelligence reports
